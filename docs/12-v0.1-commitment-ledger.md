# v0.1 承诺台账

> 基线：`docs/00-initial-need-and-situation.md` 的 v0.1 MVP 与 FR1-FR36。
> 状态规则：只有实现、自动化证据和真实环境验证同时存在时才标记为 `VERIFIED`。

## 状态定义

| 状态 | 含义 |
|---|---|
| `VERIFIED` | 实现、自动化测试和目标环境验证均完成 |
| `IMPLEMENTED_PENDING_VALIDATION` | 代码与离线测试完成，真实服务或安装包验证未完成 |
| `PARTIAL` | 仅部分能力、骨架、枚举或基础检查存在 |
| `MISSING` | 尚无可用实现 |
| `DEFERRED` | 明确不属于 v0.1，保留后续承诺 |

## 性能发布门禁

| 指标 | 门槛 | 测量方式 |
|---|---:|---|
| 窗口可交互 | `< 1s` | Tauri 首次前端 ready 标记 |
| 轻量服务冷启动 | `< 2s` | 干净 Windows VM 三次测量取最大值 |
| 空闲新增私有内存 | `< 30MB` | UI ready 后五分钟的守护进程私有内存 |
| 空闲 Python Worker | `0` | 进程树检查 |
| 重 Worker 首次 ready | `< 5s` | 首个需 Python 的 RPC 到 worker ready |
| Worker 回收 | `<= 60s` | 无 job、无待处理请求后的进程树检查 |
